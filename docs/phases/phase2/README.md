## 运行模式

```
实盘主循环：
  1. 连接WebSocket行情
  2. 接收实时数据
  3. 策略判断信号
  4. 执行下单（如有信号）
  5. 监控持仓状态
  6. 检查风控规则
  7. 循环/等待
```

---

## 告警配置示例

```yaml
alerts:
  telegram:
    bot_token: "YOUR_BOT_TOKEN"
    chat_id: "YOUR_CHAT_ID"
  
  rules:
    - 连续亏损3次 → 发送告警
    - 单日亏损超3% → 发送告警+暂停交易
    - 程序崩溃 → 发送告警
    - API错误 → 发送告警
```
